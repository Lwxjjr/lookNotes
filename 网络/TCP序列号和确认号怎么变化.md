发送的TCP报文：
1. 序列号 = ==上一次**发送的**序列号== + len（数据长度）
	- 如果上一次发送的是SYN报文或者FIN报文，改为上一次发送的序列号 + 1
2. 确认号 = ==上一次**收到的**序列号== + len（数据长度）
	- 如果收到的是SYN报文或者FIN报文，改为上一次收到的序列号 + 1

序列号：用于解决网络包乱序问题  
确认应答号：用于解决丢包问题  
控制位：用于标识TCP报文是什么类型的报文  

==第二次和第三次握手报文中的==  
==确认号 = 对方的序列号 + 1==  
==序列号 = 对方的确认号==   
1. 告诉对方，我方已经接受到了SYN报文
2. 告诉对方，我方下一次期望收到的报文的序列号为此确认号  

客户端与服务端完成 TCP 三次握手后  
**第一个TCP 数据报文的序列号和确认号 = 第三次握手的 ACK 报文中序列号和确认号**
